csub 支持哪些词法记号

“csub”是我起的名字， 表示 c 语言的子集。为了降低难度，我不打算写一个功能齐全的 C 编译器，而是写一个简易编译器，仅实现 C 语言的一部分语法特性。

本书中提到的 csub ,可能指我们定义的语言，也可能指我们的编译器。请大家根据语境区分。

csub 支持的词法记号如下表：

标识符 和 C 语言命名规则一致 最大长度可设置

关键字(流程控制) if, else, for, while, break, switch, case, default, continue, do, return
常量 十进制整数，字符，字符串 例如 987，‘A’,“hello”

需要说明的是，我们的指针仅支持指向整数变量的指针和指向字符串的指针。另外，csub 没有预处理的功能。

状态转换图

scan()函数返回的是一个个字符，如何从这些字符中识别出词法记号呢？

答案是状态转换图。

状态转换图由状态节点和转移边构成。

  • 状态节点：用圆圈表示（对于最终状态，用双圈表示）。

  • 转移边：用箭头表示。箭头上标记有字符或字符集，表示某一状态接受箭头上的字符或字符集后转换到另一状态。

  • 一张状态转换图中的状态个数是有限的，其中有一个初态，至少有一个终态。

上图是整数常量的状态转换图。这张图有 3 个圆圈，分别表示状态 1、状态 2、状态 3；

注意，这里的 1、2、3 就是为了区分 3 种不同的状态，你也可以改成别的名字。

1 是初始状态。在读入任何字符前，状态转换图总是处于初始状态。

现在结合具体场景来说明此图的含义。

假设源文件只有一行：

  1. 调用 scan() 得到字符‘1’, 这是一个数字，所以进入状态 2；
  2. 再次调用 scan() 得到字符 ‘0’，是数字，还在状态 2；
  3. 第三次调用 scan() 得到字符 ‘8’，是数字，还在状态 2；
  4. 第四次调用 scan() 得到 ‘+’，不是数字，进入状态 3；

状态 3 是双圈，表示终止状态，说明找到了一个整数常量（108）。

细心的读者会问，状态 3 上面为什么有个 “*” 呢？

这个星号表示“回退”。通俗讲就是识别出来的 108 并不包括步骤 4 中的 ‘+’。

再举个例子，上图是识别‘<’ 和 ‘<=’ 的状态转换图。

当遇到 ‘<’ 的时候，进入状态 2，若之后遇到 ‘=’，进入状态 3，这是终止状态，说明识别到了 ‘<=’；注意，状态 3 上面没有星号，表示最后遇到的这个 ‘=’ 是包括在词法记号‘<=’里面的；

假如进入状态 2 之后，遇到的不是‘=’，那么进入状态 4，这也是终止状态，说明识别到了 ‘<’；因为状态 4 上面有星号，表示最后遇到的这个其他字符不在‘<’里面。

对于每类词法记号，都可以画出类似的状态转换图，然后把图转换成代码，这样就完成词法分析了。

需要说明，终止状态也可能是错误状态，这需要在代码中做错误处理。

贪心原则

词法分析器在识别记号时，会遵循“贪心”原则，也就是尽可能读取最长的有效字符序列。

举个例子， a+++b 应该解析成 (a++) + b还是 a + (++b)?

根据贪心原则，解析完标识符 a 后，scan（） 的结果是 +，这时候需要再次 scan（），如果得到 +， 那么按 ++ 处理。如果不是 +，这个字符应该留给下一个 token；

所以，a+++b 会被解析成 (a++) + b；注意，这里面的小括号不是解析出来的结果，只是为了阅读方便我加上去的。

为了统一，除非遇到文件末尾，我们解析出 token 后会再调用一次 scan（），使 ch_g 获得新的值，下次调用 get_token() 的时候，就从这个值开始处理。
