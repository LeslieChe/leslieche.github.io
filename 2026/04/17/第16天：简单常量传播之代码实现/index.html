第16天：简单常量传播之代码实现

第16天：简单常量传播之代码实现

本节我们讲解常量传播的代码。

常量传播按照流程分为下面几个函数。

  1. const_prpgt_init
  2. simple_const_prpgt_analyse
  3. algebra_simplify
  4. cond_jmp_optimize
  5. remove_unreachable_block

第一个是初始化，包括内存分配、设定集合的初值和边界值。

第二个是使用迭代算法进行数据流值的传递与计算，直到达到不动点。

第三个是代数化简，根据每条指令的 in 和 out，尝试化简。

第四个是条件跳转优化，如果条件能化简为常量，则可以做多余边的删除。

第五个是移除不可达的块。

现在为读者仔细讲解每个函数。

const_prpgt_init

const_prpgt_info_t 这个结构体是专门为常量传播设计的。

第 10 行的 bound_vals 是一个 double 类型的指针，初始化完成后这个指针会指向数组的首元素，此数组里面是边界值。

复习一下什么是边界值。对于正向传播，Entry 块的 out 集合的初始值就是边界值。

如何确定边界值呢？对于要优化的中间代码，我们需要统计它使用到的全局变量、局部变量和形式参数。

对于全局变量，如果是非基本类型，边界值是 NAC；如果是基本类型，那要看有没有用常量初始化，如果有，边界值就是初始化的值，否则边界值为 0；这是因为后面生成汇编代码的时候，csub 会给没有初始化的全局变量都赋予 0 值。

对于形式参数，一律赋值为 NAC。

对于局部变量，规则和全局变量类似，不同的是如果没有用常量初始化，就赋值为 UNDEF。

以上规则不用纠结，为了说明原理，我们对复杂情况做了简单的处理，毕竟我们是学习编译原理，而不是造一个工业级编译器。

第 11 行的 init_vals 和第 10 行的 bound_vals 作用类似，只不 init_vals 代表的数组是存储初始值的，也就是其他 block 的 out 集合的初值。我们规定， 初始值全部是 UNDEF。

有读者问为什么不初始化 in 集合？

因为除了 Entry 块，每个基本块的 in 集合都来自于它前驱的 out 集合, 如果是单一前驱，那它的 in 集合和前驱的 out 集合是完全相等的。如果它有两个或以上的前驱，则需要用交汇函数把这些前驱的 out 合并在一起，作为它的 in;所以，对于正向分析，只需要初始化每个基本块的 out 集合。同理，对于逆向分析，只需要初始化每个基本块的 in 集合。

需要注意，第 905 行的判断是必要的，因为跳转指令的目标块也可能是紧跟它的块，如果是这样，即使删除了跳转指令，但是由于顺序执行，还会执行紧跟块，所以这种情况下当前块到目标块的边是不能删除的。

第 910~911，如果指令是“为真则跳转”且条件为真，或者指令是“为假则跳转”且条件为假，那结果是一定跳转。第 917 行，把有条件跳转指令换成无条件跳转指令。既然当前块一定会跳转到目标块，那么应该解除当前块到紧跟块的边。同理，如果目标块是紧跟块，就不能解除。

如果是相等则跳转（IR_OP_JE），那么会用到 2 个操作数，第 922 行获得 arg2 的格值，当它也是常量的时候，才能优化。第 924 表示两个操作数相等，则肯定跳转。剩下的代码和前面同理，不再解释。

如果是不相等则跳转（IR_OP_JNE），道理类似，不再解释。

因为条件跳转优化的过程可能会删除某些边，导致某些基本块不可达，所以我们调用 remove_unreachable_block 移除不可达的基本块。

至此，常量传播优化就完成了。

有的同学会问，为什么 B7 没有得到化简？按理说应该把 B2 中的 x = 2 传播到 B7,并且把 B6 的 y = 1 传播到 B7，把指令（28） 替换为 t11 = 2;

这个问题问得好。常量传播框架不止一种，例如简单常量传播、条件常量传播、稀疏常量传播等。我们学习的是简单常量传播。

对于简单常量传播，第一步是到达定值，在到达定值的基础上，做代数化简和条件跳转优化。上面的例子，当到达定值的时候，B7 的 x 和 y 的格值都是 NAC；因为 B5 和 B6 分别把 y = -1 和 y = 1 向 B7 传播，所以 y 就变成 NAC 了；B2 和 B3 分别把 x = 2 和 x = 0 向 B4 传播，所以 x 就变成 NAC 了；

既然 B7 的入口处， x 和 y 的格值都是 NAC，那自然没有办法优化。

有的读者会疑惑，不是删除了 1->3、3->4、4->5、5->7 的边吗？应该只有 B2 的 x 可以往下传播，同理，应该只有 B6 的 y 可以往下传播。我懂你的意思，但是删除边是在到达定值之后做的。你的意思是边传播边删除，简单常量传播没有这个功能，条件常量传播应该可以满足你的要求。

条件常量传播比简单常量传播的精度更高，优化能力更强。

我们再看一个简单的例子。

  • 简单常量传播：

    在 if 语句之前，是知道 x = 5 的；但是进入 if 分支后，不利用 x == 5 这个条件；还有可能因为后续路径合并（如还有 else 分支）而将 x 升级为 NAC（非常量）；

    所以，y = x + 1 无法优化为 y = 6.

  • 条件常量传播：

    在 if 语句之前，是知道 x = 5 的；进入 if 分支后，结合 x == 5，推出在此分支内，x 一定是 5，于是把 y = x + 1 优化为 y = 6.

    这就是二者的区别。
