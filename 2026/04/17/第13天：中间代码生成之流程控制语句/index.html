生成跳转指令

我们用 new_jump_inst 函数来生成条件跳转或者无条件跳转指令。

参数 op 有以下几种：

IR_OP_RET： 无返回值的函数返回；(arg1 和 arg2 用不上)

IR_OP_RET_VAL： 有返回值的函数返回 （ arg2 用不上）

IR_OP_JMP： 无条件跳转，相当于 goto (arg1 和 arg2 用不上)

IR_OP_JF： 条件为假（等于 0）则跳转 （ arg2 用不上）

IR_OP_JT： 条件为真（不等于 0）则跳转 （ arg2 用不上）

IR_OP_JE： arg1 和 arg 2 相等则跳转

IR_OP_JNE： arg1 和 arg 2 不相等则跳转

这里面的条件怎么理解呢？其实就是表达式的值

例如 while(xxx); if(xxx); for(; xxx;)等，这里面的 xxx 都是表达式，在生成中间指令的时候，会做表达式求值。

不管是函数返回还是各种跳转，返回点和跳转的目标都是标号，也就是 new_jump_inst 函数第 2 个参数 target。

switch-case 语句

先举个例子。

请问 foo(3) 会输出什么？

有的读者是这么想的：先匹配 case 1，匹配不上，然后匹配 case 2, 还是匹配不上，接下来是 default，匹配上了，所以最后输出 I am default

其实不是这样，default 在其他所有的 case 标签都匹配不上才能轮到它，和它的位置无关。所以，当 case 2 匹配不上，会判断是否匹配 case 3，匹配不上，也没有其他的 case 标签了，只能匹配 default，于是最后输出 I am case 3。

另外，还有一点要注意，一旦匹配上了某个 case/default 标签，如果没有遇到 break 或 return，会继续执行下面的 case/default 标签。

例如：

foo(4) 会输出什么？

I am default

I am case 3

因为 case 1,2,3 都匹配不上，所以执行 default 分支，输出 I am default 后，没有遇到 break 或者 return ,会继续往下执行，再输出 I am case 3；

现在可以构思中间代码了。思路是

1、和每个 case 标签对比（default 除外），一旦匹配上，就进入那个分支

2、如果所有的 case 标签都匹配不上，则看看有没有 default，有则进入 default 分支；没有则跳出 switch-case 语句；

注意，我把源码中的 case 1, case 2 等叫做 case 标签， default 关键字叫做 default 标签；如果提到“生成标签”或者“插入标签”，指的是中间代码中的标签或标号。请读者注意区分。

顺着这个思路和上面的代码，可以写出中间代码

但是仔细琢磨，就会发现问题，a 的取值 1，2，3 和标号 @case_1、@case_2、@case_3 等都是在解析 case 语句的时候才能获取的信息，不可能还没有解析到 case 语句，就把第 1~4 行的中间代码构造出来。

所以这里和前面的 for 语句面临一样的困境，就是要生成的中间代码的顺序和解析源码的顺序不一致。

如何解决呢？有了前面 for 语句的经验，很容易想到我们可以先解析 case 语句，然后把 case 语句的中间代码移动到临时链表，再生成第 1~4 行的中间代码，最后把临时链表上的中间代码移动回来。

另外，在解析 case 语句的时候，还需要做语义分析，如果发现 case 值重复了，或者出现了一个以上的 default，则报错。

break 和 continue 语句

break 可以用在循环语句及 switch-case 语句中，作用是跳出本层循环，或者跳出本层 switch 语句。

continue 可以用在循环语句中，作用是跳过当前这次循环中 continue 之后的语句，直接进入下一次循环判断（对于 while, do-while）或者迭代表达式的求值（对于 for）。

我们可以用无条件跳转实现 break 和 continue 语句的中间代码。

下面的表格列出了四种控制语句中 break 和 continue 语句如何翻译。

注意，下表格的标签名仅是一个简称，实际代码中会用 gen_name 函数生成标签，格式类似 @Lxx_exit，保证标签名不会重复。

语句类型 break（从 end 栈取标签） continue（从 begin 栈取标签）
while goto @exit goto @while
do-while goto @exit goto @do
for goto @exit goto @iter
switch-case goto @exit 非法；让 NULL 入栈

另外，不管是循环还是 switch-case 都是可以嵌套的，每一层都有自己的标签，为了让 break 和 continue 语句跳转到正确的标签，我们需要动态跟踪每一层的标签，这和我们之前跟踪大括号记录作用域很像，所以同样用栈来实现。

我们准备 2 个栈，一个叫 begin 栈，用来存放 continue 语句要跳转的标签；另一个叫 end 栈，用来存放 break 语句要跳转的标签。

开始的时候这两个栈都是空的，每当进入控制语句（上面表格中的四种之一，if-else 不算），就把一对标号入栈(准确说是标号的地址)。

当遇到 continue 语句，就获取 begin 栈栈顶的标签，生成 goto @xxx 指令；当遇到 break 语句，就获取 end 栈栈顶的标签，生成 goto @yyy 指令；

有人问 switch-case 语句中遇到 continue 怎么办？因为 continue 不能用在 switch-case 中，所以要报错。为了解决这个问题，当进入 switch-case 语句的时候， @exit 标签入 end 栈，NULL 入 begin 栈。这样当在 switch-case 语句中遇到 continue 的时候， begin 栈栈顶就是 NULL 这个特殊值，我们就知道出错了。
