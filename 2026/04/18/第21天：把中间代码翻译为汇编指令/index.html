好了，把中间代码翻译为汇编代码的通用步骤是：

1、把 arg1 送入寄存器 eax

2、如果有第二个操作数 arg2，把它送入寄存器 ecx

3、选择合适的汇编指令，完成计算，假设结果在 eax 中

4、把 eax 的值写回变量 result

除法和求余

和加减乘不同，对于除法，必须作带符号的扩展。因为除法可以看成是向右移位，高位的 1 会移动到低位，所以带符号扩展和不带符号扩展的结果完全不一样。

举个简单的例子，

char x1 = -4;

char x2 = x1 / 2;

-4 = 0xFC,如果作零扩展，就是 0x000000FC， 0x000000FC/2 = 0x7E = 126

如果作符号扩展，就是 0xFFFFFFFC, 0xFFFFFFFC/2 = 0xFFFFFFFE = -2

idiv 指令的格式如下表

操作数大小 被除数位置 除数（操作数） 商 余数
8 位 AX r/m8 AL AH
16 位 DX:AX r/m16 AX DX
32 位 EDX:EAX r/m32 EAX EDX
64 位 RDX:RAX r/m64 RAX RDX
  • 被除数必须是操作数宽度的两倍
  • 高半部分放 DX/EDX/RDX，低半部分放 AX/EAX/RAX

setg、setge、setl、setle 等属于条件设置指令（Set Byte on Condition）

属于 SETcc 指令家族。它们根据 CPU 标志位（EFLAGS/RFLAGS）将目标操作数设为 0 或 1

注意：操作数必须是 8 位（寄存器如 al、bl，或内存操作数）

setge Greater or Equal 大于等于 SF=OF a >= b
setl Less 小于 SF≠OF a < b
setle Less or Equal 小于等于 ZF=1 or SF≠OF a <= b

第 449 表示没有返回值的返回，就是 return;

第 454 行，表示带有返回值的，例如 return x;

在生成中间代码的时候，我们为每个函数都生成了一个 return 标号，位于函数的末尾。

第 1168 行，第 1174 行，在创建 IR_OP_RET 和 IR_OP_RET_VAL，已经把这个 return 标号设置到了 inst->jmp_target 字段。

所以 inst->jmp_target->label 就是跳转的标号。

根据 cdecl 调用约定，函数返回值用 eax 传递。

如果返回值是 char 类型且函数声明的返回值也是 char，则把 al 零扩展到 eax，这样做是为了安全。

如果返回值是 char 类型，但是函数声明的返回值类型是 int，则把 al 带符号扩展到 eax.
